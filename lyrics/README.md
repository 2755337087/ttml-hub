# 歌词目录与命名规范

推荐路径：

```text
lyrics/<主艺术家>/<歌名> [<8位稳定ID>].ttml
```

示例：`lyrics/周杰伦/晴天 [a1b2c3d4].ttml`

- 文件与目录统一使用 UTF-8 NFC，可保留中文、日文、韩文等原始文字。
- 不使用 `\ / : * ? " < > |`；创建脚本会自动替换这些字符。
- ID 是 8 位小写十六进制，创建后永久保持不变。编辑歌词、改歌名或移动目录时不要生成新 ID。
- 同名歌曲、现场版、伴奏版分别使用不同 ID，并在歌名中标注版本。
- 可增加同名 `.meta.json`，它会覆盖从路径推断的歌名和艺术家，并提供专辑、语言、别名、ISRC 与授权来源。

创建歌词：

```bash
npm run lyric:new -- --title "晴天" --artist "周杰伦"
```

提交前检查：

```bash
npm run lyrics:check
```

公开仓库不等于歌词自动获得开放授权。请只提交你有权分享的内容，并在 `.meta.json` 中填写 `license` 和 `sourceUrl`。
